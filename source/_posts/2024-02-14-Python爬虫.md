有段时间没写爬虫了，最近想重新捡起来。起因是想爬一下QQ空间的历史说说，做一些数据分析的练习。翻了翻以前的代码，发现很多库的用法已经变了，干脆从头梳理一遍。

## 先说框架选型

Python爬虫这块，主流的工具就那么几个：requests + BeautifulSoup、Scrapy、Selenium。三者定位不同，选哪个取决于你要爬什么。

### requests + BeautifulSoup

最轻量的组合。requests负责发HTTP请求，BeautifulSoup负责解析HTML。适合爬静态页面，或者你能拿到API接口的场景。

```python
import requests
from bs4 import BeautifulSoup

url = "https://example.com"
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}
resp = requests.get(url, headers=headers)
soup = BeautifulSoup(resp.text, "lxml")
title = soup.find("title").text
```

几行代码就能跑起来，调试也方便。我的经验是，大多数爬虫需求，先用requests试试，很多时候页面直接给你返回HTML，根本不需要渲染JS。

### Scrapy

Scrapy是个完整的爬虫框架，自带调度器、中间件、管道这些东西。优点是快，异步架构处理大量页面效率很高。缺点是项目结构比较重，写个简单的爬虫反而不如直接用requests方便。

我之前爬过一个小说网站，用Scrapy写了完整的spider，包括去重、限速、数据导出CSV，跑起来确实舒服。但如果是像QQ空间这种需要登录、有复杂反爬机制的，Scrapy本身的登录处理不太方便，通常还是要配合Selenium用。

### Selenium / Playwright

这类工具的核心能力是操控真实浏览器。页面是JS动态渲染的、需要登录的、有验证码的——这些场景用Selenium或Playwright比较合适。

```python
from selenium import webdriver

driver = webdriver.Chrome()
driver.get("https://example.com")
# 等待页面加载
element = driver.find_element("css selector", ".data-content")
print(element.text)
driver.quit()
```

缺点也明显：慢。我测试过同一个页面，requests抓取大概3秒，Selenium要40多秒。因为它要启动一个完整的浏览器进程，等JS执行完再拿DOM。

### 我的选择

日常爬虫：requests + BeautifulSoup，够用就行。
大规模批量采集：Scrapy，性能好。
需要登录/JS渲染：Selenium或Playwright。

实际上很多时候是组合使用的。比如我这次爬QQ空间，就是Selenium登录拿cookie，然后用requests去请求数据接口。

## 实战：QQ空间爬虫

QQ空间是个挺好的练习目标，因为它基本集齐了爬虫会遇到的所有难题。

### 第一步：登录

QQ空间的登录页面嵌在一个iframe里，而且有滑块验证。纯requests是搞不定的，直接上Selenium。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
import time

driver = webdriver.Chrome()
driver.get("https://user.qzone.qq.com/")

# QQ空间登录页嵌在iframe里，要切换进去
driver.switch_to.frame("login_frame")
driver.find_element(By.ID, "switcher_plogin").click()
time.sleep(0.5)

driver.find_element(By.ID, "u").send_keys("你的QQ号")
driver.find_element(By.ID, "p").send_keys("你的密码")
driver.find_element(By.ID, "login_button").click()

# 等待登录完成
time.sleep(5)
```

这里有个坑：登录成功后页面会跳转，但Selenium不会自动等待。我一开始没加sleep，直接去取cookie，结果取到的是登录前的空cookie。加了显式等待之后才稳定下来。

### 第二步：获取g_tk

QQ空间的API请求需要一个叫g_tk的参数，它是从cookie里的p_skey值经过一个哈希算法算出来的。

```python
def get_g_tk(p_skey):
    hashes = 5381
    for letter in p_skey:
        hashes += (hashes << 5) + ord(letter)
    return hashes & 0x7fffffff
```

这个算法本身不难，难的是你知道它存在。我一开始根本不知道有这个参数，请求一直返回403，debug了挺久。后来搜了相关资料才发现，QQ空间的请求必须带上g_tk。

除了g_tk，还需要从页面源码里提取qzonetoken。这个token藏在一段内联JS代码里，用正则可以拿到。

```python
import re

html = driver.page_source
pattern = r'window\.g_qzonetoken = \(function\(\)\{ try\{return \"(.*)\";'
match = re.findall(pattern, html)
qzonetoken = match[0] if match else ""
```

### 第三步：请求数据接口

拿到cookie、g_tk和qzonetoken之后，就可以用requests去请求QQ空间的数据API了。

```python
import requests
import json

session = requests.Session()
# 把Selenium拿到的cookie注入requests
for cookie in driver.get_cookies():
    session.cookies.set(cookie["name"], cookie["value"])

url = "https://user.qzone.qq.com/proxy/domain/m.qzone.qq.com/cgi-bin/new/get_msgb"
params = {
    "uin": "目标QQ号",
    "hostUin": "目标QQ号",
    "start": 0,
    "s": "0.1",
    "format": "jsonp",
    "num": 10,
    "inCharset": "utf-8",
    "outCharset": "utf-8",
    "g_tk": g_tk,
    "qzonetoken": qzonetoken,
}
resp = session.get(url, params=params)
```

返回的是JSONP格式的JSON数据，需要去掉回调函数的包装才能解析。

### 第四步：处理反爬

QQ空间的反爬主要体现在几个方面：

**登录验证**：滑块验证，Selenium能处理但需要手动过一次，之后cookie就保存了。

**请求频率限制**：请求太快会被封。我一开始没加延迟，跑了没多久就被限制了。后来加了随机的sleep，1到3秒随机间隔。

```python
import time
import random

time.sleep(random.uniform(1, 3))
```

**动态加载**：QQ空间的内容是通过AJAX接口加载的，不是直接写在HTML里的。所以解析HTML是拿不到内容的，必须找到API接口。

**JSONP包装**：返回的数据被包在回调函数里，需要截取。我的处理方式是取掉前10个字符和后3个字符（去掉`callback(`和`);`）。

```python
text = resp.text[10:-3]
data = json.loads(text)
```

### 第五步：数据清洗

留言板的数据是HTML格式的，里面混着各种标签、script、style。需要写一个清洗函数把HTML标签去掉。

```python
import re

def clean_html(html_str):
    # 去掉script和style
    html_str = re.sub(r'<script.*?</script>', '', html_str, flags=re.S)
    html_str = re.sub(r'<style.*?</style>', '', html_str, flags=re.S)
    # 去掉HTML标签
    html_str = re.sub(r'<.*?>', '', html_str)
    # 处理HTML实体
    html_str = html_str.replace('&nbsp;', ' ')
    html_str = html_str.replace('&lt;', '<')
    html_str = html_str.replace('&gt;', '>')
    html_str = html_str.replace('&amp;', '&')
    return html_str.strip()
```

## 遇到的问题

总结几个值得注意的地方：

1. **Cookie过期**：QQ空间的cookie有效期不长，跑一会儿就要重新登录。我后来写了个检测机制，发现请求失败就自动重新登录。

2. **iframe切换**：Selenium操作QQ空间必须先切换到iframe里，不然找不到登录元素。操作完再切回来。

3. **编码问题**：中文内容经常出现乱码，requests返回的encoding有时候不对，手动设置`response.encoding = "utf-8"`能解决大部分问题。

4. **页面结构变了**：QQ空间的HTML结构时不时会变，之前写的选择器过一段时间就失效了。所以解析逻辑最好写得宽泛一点，别依赖太细的层级结构。

5. **robots.txt**：虽然QQ空间的robots.txt没有明确禁止爬取，但还是要控制频率，不要给别人服务器造成负担。

## 小结

重新写了一遍爬虫，发现和几年前比，基本原理没变，但工具和生态变化不少。requests依然好用，Selenium从2.x升级到了4.x，Playwright成了新的选择，Scrapy也出了很多插件。

对于QQ空间这种有复杂反爬的站点，单靠一个库是搞不定的。Selenium解决登录，requests解决请求效率，正则+JSON解析解决数据提取，三者配合才能跑通。

下次打算试试Playwright替代Selenium，据说在等待策略和性能上都比Selenium好一些。
