## 为什么需要微调

<!-- more -->

预训练模型（如 LLaMA、Qwen、ChatGLM 等）在海量文本上学习了通用的语言知识，但这些知识是"泛化"的。当你想让模型在特定领域表现更好——比如回答中文百科知识、遵循特定格式、或者适配某个行业场景——预训练模型的表现往往不够用。

全量微调（Full Fine-Tuning）是直接更新模型的所有参数。听起来很直接，但问题在于资源消耗。以 LLaMA-2-7B 为例：

- FP16 权重：14GB
- 梯度：14GB
- AdamW 优化器状态：28GB

仅训练状态就需要 56GB 以上的 GPU 显存。而 Google Colab 免费版提供的是 T4 GPU，显存只有 16GB。更不用说 70B 参数的模型需要 560GB，得上 8 张 A100。

这就催生了参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）的研究方向：能否只训练极少量的参数，就让模型学会新任务？LoRA 就是这个方向上最成功的方案之一。

## LoRA 的核心原理

LoRA 的全称是 Low-Rank Adaptation，低秩自适应。它的核心假设是：微调过程中的权重更新矩阵具有很低的"内在秩"（intrinsic rank）。

具体来说，对于预训练模型中的一个权重矩阵 W₀ ∈ R^{d×k}，全量微调会学习一个更新 ΔW ∈ R^{d×k}。LoRA 的做法是将 ΔW 分解为两个低秩矩阵的乘积：

```
W = W₀ + B × A
# B ∈ R^{d×r}, A ∈ R^{r×k}
# 可训练参数 = d×r + r×k = r×(d+k)
# 当 r << min(d, k) 时，参数量大幅减少
```

举个例子，LLaMA-7B 的注意力层 d=k=4096：

- 全量微调：4096 × 4096 = 16,777,216 参数
- LoRA (r=16)：4096 × 16 + 16 × 4096 = 131,072 参数
- 压缩比：128 倍

前向传播时，LoRA 的更新会叠加到原始权重上：

```python
h = W₀ @ x + (B @ A) @ x * (alpha / r)
```

其中 alpha/r 是缩放因子，alpha 通常设为 2r。

## LoRA 的关键超参数

| 超参数 | 推荐范围 | 作用 |
|--------|----------|------|
| rank (r) | 8 - 64 | 控制表达能力与参数量。简单任务用 8-16，复杂任务用 32-64 |
| lora_alpha | 16 - 128 | 控制更新幅度。通常设为 2r 或固定 16 |
| target_modules | q,k,v,o 或全部线性层 | 决定注入 LoRA 的层 |
| lora_dropout | 0.0 - 0.1 | 正则化，小数据集建议 0.05-0.1 |
| learning_rate | 1e-4 - 3e-4 | 比全量微调高 5-10 倍 |

## QLoRA：4-bit 量化 + LoRA

LoRA 减少了可训练参数，但冻结权重仍然需要加载到 GPU 显存中。LLaMA-7B 的冻结权重以 FP16 存放需要 14GB——这已经超出 Colab 免费版的安全范围。

QLoRA（Quantized LoRA）解决了这个问题：将冻结的基础模型量化到 4-bit，然后在其上注入 LoRA adapter 进行 16-bit 精度的微调。

QLoRA 的三个关键技术：

1. **NF4（NormalFloat4）**：专为正态分佈权重量化的 4-bit 数据格式，比传统 INT4/FP4 误差低 10-30%
2. **双重量化**：对量化常数再量化，将开销从 0.5 bit/param 降至 0.127 bit/param
3. **分页优化器**：利用 CPU 内存作为缓冲，防止长序列训练时 OOM

内存对比（LLaMA 模型）：

| 方法 | 7B | 13B | 33B |
|------|-----|------|------|
| 全量微调 FP16 | ~56 GB | ~104 GB | ~264 GB |
| LoRA FP16 | ~18 GB | ~32 GB | ~72 GB |
| QLoRA 4-bit | ~6 GB | ~10 GB | ~22 GB |

QLoRA 的突破性意义在于：7B 模型可以在免费 Google Colab T4 上完成微调。

## 实战：在 Colab 上用中文维基百科微调模型

### 环境准备

Google Colab 免费版提供的 T4 GPU 有 16GB 显存，足以运行 QLoRA 微调一个 7B 模型。

首先安装必要的库：

```python

```

### 从 ModelScope 下载中文维基百科数据集

ModelScope 是阿里的开源模型平台，提供了大量中文数据集。中文维基百科数据集 `wikipedia-cn-20230720-filtered` 包含了经过清洗的中文维基百科内容。

```python
from modelscope import snapshot_download

# 下载数据集
dataset_path = snapshot_download(
    'AI-ModelScope/wikipedia-cn-20230720-filtered',
    cache_dir='./data'
)
```

数据集下载后需要做一些预处理，将原始文本转换为指令微调格式。维基百科数据通常是纯文本段落，我们需要将其构造为问答对或指令格式。

### 加载模型（4-bit 量化）

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig

# 4-bit 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
)

# 加载模型和分词器
model_id = "Qwen/Qwen2.5-7B-Instruct"  # 或其他适合的中文模型
tokenizer = AutoTokenizer.from_pretrained(model_id, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    quantization_config=bnb_config,
    device_map="auto",
    trust_remote_code=True
)
```

### 注入 LoRA

```python
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training

# 准备模型进行量化训练
model = prepare_model_for_kbit_training(model)

# LoRA 配置
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)

# 查看可训练参数量
model.print_trainable_parameters()
# 输出类似：trainable params: 41,943,040 || all params: 7,614,556,160 || trainable%: 0.55%
```

### 训练配置与执行

```python
from trl import SFTTrainer
from transformers import TrainingArguments

training_args = TrainingArguments(
    output_dir="./output",
    per_device_train_batch_size=1,
    gradient_accumulation_steps=8,
    learning_rate=2e-4,
    num_train_epochs=3,
    logging_steps=10,
    save_steps=100,
    warmup_ratio=0.1,
    fp16=True,
    optim="paged_adamw_8bit",
    gradient_checkpointing=True,
    report_to="none",
)

trainer = SFTTrainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    tokenizer=tokenizer,
    max_seq_length=512,
)

trainer.train()
```

### 保存与合并 LoRA 权重

训练完成后，LoRA adapter 的大小通常只有几十 MB，远小于完整的模型权重。

```python
# 保存 LoRA adapter
model.save_pretrained("./lora_adapter")

# 如果需要合并回基础模型（可选）
from peft import PeftModel

base_model = AutoModelForCausalLM.from_pretrained(
    model_id,
    torch_dtype=torch.bfloat16,
    device_map="auto"
)
merged_model = PeftModel.from_pretrained(base_model, "./lora_adapter")
merged_model = merged_model.merge_and_unload()
merged_model.save_pretrained("./merged_model")
```

## 训练过程中的注意事项

**数据格式**：中文维基百科的原始数据需要转换为指令格式。比如可以构造如下问答对：

```
instruction: 请介绍关于{主题}的知识
input: （可选的上下文）
output: {维基百科段落内容}
```

**序列长度**：在 T4 GPU 上，max_seq_length 建议设为 512 或 1024。太长会占用过多显存，太短会截断有用信息。

**批次大小**：free tier GPU 上通常只能用 batch_size=1，通过 gradient_accumulation_steps 来等效增大有效批次。

**训练时长**：以中文维基百科数据集为例，在 T4 上微调 7B 模型，处理几万条数据大概需要 1-2 小时。

## 效果评估

微调完成后，可以通过几个维度来评估效果：

1. **困惑度（Perplexity）**：在保留的测试集上计算模型的困惑度
2. **生成质量**：给定中文问题，观察模型回答的专业性和准确性
3. **指令遵循**：检查模型是否能正确理解并执行用户指令

实际测试中，微调后的模型在中文知识问答任务上的表现会有明显提升，尤其是与训练数据领域相关的知识。

## 总结

LoRA 的价值在于让"微调"这件事从需要多张高端 GPU 的工作，变成了在免费 Colab 上就能完成的实验。QLoRA 进一步将显存需求压到 4-bit，让 7B 模型在 16GB 显存的 T4 上也能跑起来。

对于中文场景，从 ModelScope 下载中文维基百科数据集，用 LoRA 在 Colab 上微调，整个流程从数据准备到模型训练可以在一个下午内完成。这是当前成本最低、门槛最低的大模型定制化方案。
